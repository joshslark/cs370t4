Questions for Client

The Outsourcers

Creation Date : 02/10/2015

Last revision : 07:23pm, 02/15/2015

Person responsible for document : Rohan Khakurel

Statement of purpose :

This document was created to formulate a list of questions for Andrea Grazzini that will be used during our interview with her on Tuesday, February 17, 2015. Our team, The Outsourcers, met in Room G on the Second Floor of Pickler Memorial Library on February 10, 2015 to discuss and organize questions we have about her idea for a software project: a bubble map of civic engagement gaps in the United States. With the answers to these questions, we plan to formulate a requirements document that details exactly what is expected from the software system.

A. Background Image

  • 1. Should the starting image be a blank map of the US?
  • 2. What are the set of colors for the basic map?
  • 3. Should the user be able to designate a color for the inside and outside of a border?
  • 4. Should the user be able to modify the border appearance?
  • 5. What textures should be included and how would they enhance the map?
  • 6. What kind of topographical views are necessary and how would they enhance the map?

B. Data Visuals

  • 1. What are the set of colors for the bubbles and lines?
  • 2. What does it mean to be merge-able?
  • 3. What does the term wave-over description imply?
  • 4. Should the user be able to layer the data set filters or only show one at a time
  • 5. Should the user have options to include only node wave-over descriptions or line wave-over descriptions?

C. Key

  • 1. What is in the key?
  • 2. Should the key be interactive?
  • 3. Is the key separate from the available filters?

D. Predictive/Image Data

  • 1. What methods should be used to predict data?
  • 2. How will the core data be provided?
  • 3. How should new data be added?
  • 4. Should the stored data be static or automatically updatable?

E. Miscellaneous

  • 1. Who is the target audience?
  • 2. Who will directly access the map?
  • 3. Where will the map be available?
  • 4. Does the software have to be able to record itself?
  • 5. Is this a web app or a Windows program or both?
  • 6. How will videos be used?
